## Design Context

### Users
Pylon serves developers across all experience levels — solo devs, team members, and power users/tinkerers. They reach for Pylon as a daily AI coding companion embedded in their workflow. Their context is focused deep work: reading code, debugging, reviewing PRs, and iterating on features. They expect a native-feeling desktop experience, not a web app in a frame.

### Brand Personality
**Precise, refined, deliberate.** Every detail is considered — nothing is accidental. Pylon communicates through restraint: surfaces, typography, and spacing do the work, not color or ornamentation. It should feel like a precision instrument — sharp, reliable, and crafted with intent. Confidence comes from what's left out as much as what's included.

### Aesthetic Direction
- **Visual tone:** Neutral dark — true neutral or slightly cool darks, closer to Linear/Vercel. Cleaner and more modern than the previous clay-tinted warmth. No pure black.
- **Palette philosophy:** Pure monochrome. The interface is grayscale — the accent is achromatic (`#ececed`, white/near-white). Color appears only through semantic states (success, error, warning, info, special). Primary buttons are inverted: light bg, dark text.
- **Typography:** Space Grotesk for UI text (geometric, clean), Instrument Serif for rare display moments (warmth, character — used sparingly), JetBrains Mono for code and data.
- **Density:** Context-dependent (see Density Modes below).
- **Motion:** Subtle and purposeful — staggered fade-ups on entrance, gentle scale on hover/tap, `easeOutQuart` easing. Never bouncy or attention-seeking.
- **Icons:** Lucide React at 18px for nav, 13-16px inline. Thin stroke weight matches the refined aesthetic.
- **Theme:** Dark mode only. No light mode.
- **References:** Linear 2024+ (dense, fast, keyboard-driven, crisp), Vercel / v0.dev (minimal, monochrome-forward, developer-oriented), Raycast (command-palette UX, restrained color).
- **Anti-references:** ChatGPT (generic AI chat bubbles), VS Code (heavy IDE chrome), Electron jank (sluggish, web-pretending-to-be-native).

### Design Principles

1. **Precision through restraint** — Every pixel is deliberate. Color is earned, not given freely. Let surfaces, type, and spacing communicate hierarchy. If removing an element doesn't hurt, remove it.

2. **Information density, contextually** — Working views (diffs, code, tool output) are dense and data-rich. Navigation, empty states, and landing views breathe. Never one-size-fits-all.

3. **Native-grade interaction** — Instant response, no layout shift, no loading spinners where a skeleton or transition will do. The app must feel faster than the web.

4. **Quiet confidence** — No gratuitous animations, no exclamation marks in the UI, no "magic" language. The tool does the work; the interface reports it calmly.

5. **Keyboard-first, mouse-friendly** — Every action reachable by keyboard (Cmd+N, Cmd+1-9, command palette). Mouse interactions are equally polished but never required.

### Density Modes

Two explicit density contexts govern spacing and layout decisions:

**Working mode** — For diffs, code blocks, tool output, PR reviews, agent flow visualization:
- Tight spacing: `gap-1` to `gap-2`, `px-2 py-1` to `px-3 py-2`
- Smaller text: `text-xs` (12px), `text-[11px]`, `text-[10px]` for metadata
- Minimal padding around content — let the information pack
- Collapse non-essential UI; expand on hover/click

**Navigating mode** — For landing pages, empty states, sidebar navigation, modals, settings:
- Generous spacing: `gap-3` to `gap-6`, `p-4` to `p-6`
- Standard text: `text-sm` (14px) body, `text-base` (16px) for emphasis
- Breathing room between groups; clear visual separation
- Display typography (Instrument Serif) can appear here

### Spacing & Radius Conventions

**Spacing scale (Tailwind units):**
| Token | px | Usage |
|-------|----|-------|
| `0.5` | 2 | Tight icon-text pairs, inline badge padding |
| `1` / `1.5` | 4-6 | Inline element gaps, small control padding |
| `2` / `2.5` | 8-10 | List item gaps, button padding, standard inline spacing |
| `3` | 12 | Card section gaps, medium component padding |
| `4` | 16 | Standard card/panel padding, section separation |
| `5` | 20 | Modal padding, generous component spacing |
| `6` | 24 | Page-level section separation |
| `8`+ | 32+ | Hero spacing, empty state vertical centering |

**Border radius scale:**
| Token | Usage |
|-------|-------|
| `rounded` (4px) | Tiny inline badges, code snippets |
| `rounded-md` (6px) | Small controls, tab buttons, icon buttons |
| `rounded-lg` (8px) | Cards, list items, standard buttons, input areas, tool output |
| `rounded-xl` (12px) | Modals, command palette, popovers, slide-over panels |
| `rounded-2xl` (16px) | Input bar container (main chat input) |
| `rounded-full` | Pills, status dots, badges, circular icon buttons |

**Rule:** Elevation correlates with radius — higher z-index surfaces use larger radii.

**Shadow/elevation scale:**
| Level | Class | Usage |
|-------|-------|-------|
| 0 — Flat | none | Inline elements, list items, cards within panels |
| 1 — Raised | `shadow-lg` | Small dropdowns, tooltips |
| 2 — Floating | `shadow-xl` | Dropdown menus, popovers, context menus |
| 3 — Modal | `shadow-2xl` | Modals, slide-over panels, overlay dialogs |
| 4 — Commanding | `shadow-[0_20px_60px_rgba(0,0,0,0.5)]` | Command palette (highest z-index surface) |

### Component Patterns

**Overlay/Modal:**
- Backdrop: `fixed inset-0 bg-black/60 backdrop-blur-[2px]`
- Container: `rounded-xl border border-base-border bg-base-surface shadow-2xl`
- Animate with `AnimatePresence` + `motion.div` (enter: fade+scale, exit: fade)
- Always trap focus; close on Escape

**Slide-over panel:**
- `fixed top-0 right-0 bottom-0 z-50 border-l bg-base-bg shadow-2xl`
- Width: `w-[70vw] min-w-120 max-w-225`
- Animate with `motion.div` (slide from right)

**Card:**
- Standard: `rounded-lg border border-base-border-subtle bg-base-surface/50 p-4`
- Interactive: add `transition-colors hover:bg-base-raised/60 cursor-pointer`
- No shadow — flat within panels

**List item (interactive):**
- `rounded-lg px-2 py-2 text-left transition-colors hover:bg-base-raised/60`
- Active/selected: `bg-base-raised`
- Icon + content + metadata layout with `gap-2`

**Button — Primary:**
- `rounded-lg bg-accent px-3.5 py-1.5 font-medium text-base-bg text-xs transition-colors hover:bg-accent-hover`
- Inverted: light background, dark text. Active press: `active:scale-[0.98]`

**Button — Secondary:**
- `rounded-lg border border-base-border px-4 py-2.5 text-base-text-secondary text-sm transition-colors hover:bg-base-raised hover:text-base-text`

**Button — Ghost:**
- `rounded-md p-1 text-base-text-muted transition-colors hover:bg-base-raised hover:text-base-text`

**Badge/Pill:**
- `rounded-full bg-base-raised px-1.5 py-0.5 text-[11px] text-base-text-muted`
- Semantic: use `bg-error/10 text-error`, `bg-success/10 text-success`, etc.

**Empty state:**
- Centered: `flex flex-col items-center justify-center py-12 text-center`
- Icon above text (Lucide, `text-base-text-faint`)
- Text: `text-base-text-faint text-xs` or `text-sm`
- No illustrations — keep it typographic

**Tab bar:**
- `rounded-md px-2 py-1 text-[10px] font-medium`
- Active: opaque background + primary text color
- Inactive: no background + muted text

**Input:**
- `rounded-2xl border border-base-border/60 bg-base-surface transition-colors focus-within:border-accent/40`
- Border brightens on focus (accent tint), not a glow ring

**Dropdown/Popover:**
- Container: `rounded-lg border border-base-border bg-base-raised py-1 shadow-xl`
- Items: `px-3 py-1.5 text-sm hover:bg-base-border/50`

### Technical Constraints
- Tailwind CSS 4 with `@theme` custom properties (not arbitrary values where tokens exist)
- Motion library (not raw CSS transitions) for coordinated animation
- Lucide React for all iconography
- Color tokens: `base-bg`, `base-surface`, `base-raised`, `base-border`, `base-border-subtle`, `base-text`, `base-text-secondary`, `base-text-muted`, `base-text-faint`, `accent`, `accent-hover`, `accent-muted`, `accent-text`
- Semantic tokens: `success`, `error`, `warning`, `info`, `special`
- Accessibility: WCAG AA contrast ratios, `prefers-reduced-motion` respected, visible focus indicators on keyboard nav
