## Design Context

### Users
Pylon serves developers across all experience levels — solo devs, team members, and power users/tinkerers. They reach for Pylon as a daily AI coding companion embedded in their workflow, not as a novelty. They value control, speed, and tools that stay out of the way until needed. Their context is focused deep work: reading code, debugging, reviewing PRs, and iterating on features. They expect a native-feeling desktop experience, not a web app in a frame.

### Brand Personality
**Elegant, calm, intelligent.** Pylon exudes quiet confidence — it doesn't shout, doesn't clutter, and never feels anxious. The warm terracotta palette and serif display type give it a distinctly crafted character that separates it from the sea of generic AI tools. It should feel like a well-made instrument: precise, reliable, and a pleasure to hold.

### Aesthetic Direction
- **Visual tone:** Warm dark — clay-tinted neutrals (`#1a1714` base), never pure black or cold gray. Terracotta accent (`#a85838`) used sparingly for emphasis.
- **Typography:** Space Grotesk for UI text (geometric but friendly), Instrument Serif for display moments (warmth, character), JetBrains Mono for code and data.
- **Density:** Information-dense where it matters (PR diffs, code blocks, tool output) but breathing room on landing/empty states. Follow Linear's approach: pack information without feeling cramped.
- **Motion:** Subtle and purposeful — staggered fade-ups on entrance, gentle scale on hover/tap, `easeOutQuart` easing. Never bouncy or attention-seeking.
- **Icons:** Lucide React at 18px for nav, 13-16px inline. Thin stroke weight matches the refined aesthetic.
- **Theme:** Dark mode only. No light mode.
- **References:** Linear (dense, polished, keyboard-first), Raycast (warm dark theme, command-palette UX), Intent by Augment (AI coding tool aesthetic).
- **Anti-references:** ChatGPT (generic AI chat bubbles), VS Code (heavy IDE chrome), Electron jank (sluggish, web-pretending-to-be-native).

### Design Principles

1. **Warm precision** — Every pixel is deliberate, but the palette and typography keep it human. Clay over chrome, serif over sterile.

2. **Information density without clutter** — Show what matters, hide what doesn't. Tool output collapses; code blocks highlight; status is glanceable. No empty decorative space in working views.

3. **Native-grade interaction** — Instant response, no layout shift, no loading spinners where a skeleton or transition will do. The app must feel faster than the web.

4. **Quiet confidence** — No gratuitous animations, no exclamation marks in the UI, no "magic" language. The tool does the work; the interface reports it calmly.

5. **Keyboard-first, mouse-friendly** — Every action reachable by keyboard (Cmd+N, Cmd+1-9, command palette). Mouse interactions are equally polished but never required.

### Technical Constraints
- Tailwind CSS 4 with `@theme` custom properties (not arbitrary values where tokens exist)
- Motion library (not raw CSS transitions) for coordinated animation
- Lucide React for all iconography
- Color tokens: `base-bg`, `base-surface`, `base-raised`, `base-border`, `base-border-subtle`, `base-text`, `base-text-secondary`, `base-text-muted`, `base-text-faint`, `accent`, `accent-hover`, `accent-muted`, `accent-text`
- Semantic tokens: `success`, `error`, `warning`, `info`, `special`
- Accessibility: WCAG AA contrast ratios, `prefers-reduced-motion` respected, visible focus indicators on keyboard nav
